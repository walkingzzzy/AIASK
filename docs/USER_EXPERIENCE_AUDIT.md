### 3.2 返回模拟数据的功能

以下功能在模块加载失败时返回假数据，用户可能被误导：

```
1. 集合竞价分析 - /api/call-auction/stock/{code}
   返回: "模块未加载，返回模拟数据"
   
2. 机会检测 - OpportunityDetector._get_mock_stock_data()
   返回: 随机生成的股票数据
   
3. 两融数据 - MarginAnalyzer._mock_market_margin()
   返回: 随机生成的融资融券数据
   
4. 回测数据 - AKShareDataFeed._mock_data()
   返回: 随机生成的K线数据
```

**建议**: 当返回模拟数据时，应明确告知用户，并提供解决方案。

## 四、用户流程问题

### 4.1 安装流程（对普通用户过于复杂）

当前安装步骤：
```bash
1. 安装 Python 3.12+
2. 安装 uv 包管理器
3. 克隆代码仓库
4. 运行 uv sync
5. 配置 .env 文件
6. 启动后端服务
7. 启动前端服务
```

**问题**: 普通炒股用户不会使用命令行

**建议**:
- 提供一键安装包（Windows .exe / macOS .dmg）
- 或提供在线版本（Web应用）
- 或提供详细的图文安装教程

### 4.2 首次使用流程

当前状态：无引导，用户直接进入主界面

**建议添加**:
1. 欢迎页面 - 介绍系统功能
2. 风险提示 - 投资有风险声明
3. 偏好设置 - 选择投资风格、关注板块
4. 功能导览 - 高亮介绍主要功能
5. 示例演示 - 展示如何分析一只股票

### 4.3 错误处理流程

当前状态：显示技术错误信息

```javascript
// 当前错误提示
message.error('请求失败，请检查API服务是否启动')
```

**问题**: 用户不知道什么是"API服务"

**建议改进**:
```javascript
// 用户友好的错误提示
message.error('网络连接失败，请检查网络后重试')
// 并提供重试按钮和帮助链接
```

## 八、总结

当前项目从技术角度已经比较完善，但从普通炒股用户的角度来看，存在以下核心问题：

1. **入门门槛高** - 安装复杂，缺少引导
2. **理解成本高** - 专业术语多，缺少解释
3. **信任度不足** - 模拟数据未明确标识
4. **帮助不足** - 出错后无处求助

建议优先解决用户文档和新手引导问题，让普通用户能够顺利使用系统。
